== 4. UI Polish and Architecture Refinements (Current Session)
* *Time Input Localization:* Replaced custom time inputs with native browser `<input type="time">`. Removed the manual 24-hour format toggle checkbox in favor of cleanly auto-detecting the user's preferred format using `navigator.language` and `Intl.DateTimeFormat` dynamically within `TimeFormatContext`.
* *Segment Time Constraints:* Enforced a strict business rule where all profile time segments (`basal`, `icr`, and `isf`) must logically start at `00:00`.
  * Added strict backend domain validation in `Profile.kt` (`validateTimeSegments`).
  * Added matching frontend Zod `.refine()` schema validation rules in `ProfileEditor.tsx` to stop invalid UI submissions instantly.
* *Database Migrations (Liquibase):*
  * Finalized the `insulins` database table via Liquibase instead of backend application logic.
  * Enforced separation of concerns by placing the schema definition inside `001-initial-schema.yaml` and moving all initial data seeds (e.g., Novolog, Humalog, Fiasp) into a separate `002-initial-data.yaml` migration file.
  * Removed outdated manual database seeding logic from `DatabaseFactory.kt` to make Liquibase the single source of truth.
